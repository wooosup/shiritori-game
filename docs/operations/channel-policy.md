# 운영자용 채널 전환 정책 (App-only)

## 1) 전환 일정 확정
- **App-only 전환일(D-day)**: **2026-04-01 (수)**
- **웹 축소일**: **2026-03-18 (수)**

> 웹 축소일 이후에는 웹 기능을 단계적으로 제한하고, D-day부터는 신규/기존 사용자 모두 앱 채널을 기본 경로로 안내한다.

## 2) 웹 유지 범위 결정
- **결정안**: **다운로드 랜딩 페이지만 유지**
- **종료 범위**:
  - 웹 로그인
  - 웹 플레이(게임 시작/진행)
  - 웹 내 사용자 설정/프로필 편집
- **유지 범위**:
  - 앱스토어/플레이스토어 다운로드 링크
  - 최소한의 서비스 안내(점검 공지 링크 포함)

## 3) 핵심 KPI 정의

### KPI-1. 앱 로그인 성공률
- **정의**: `앱 로그인 성공 건수 / 앱 로그인 시도 건수 * 100`
- **목표**: **99.0% 이상 (일 단위)**
- **경보 기준**: **98.0% 미만 30분 지속 시 알림**

### KPI-2. 1일 리텐션(D1)
- **정의**: `D0 신규 설치 사용자 중 D1 재방문 사용자 비율`
- **목표**: **30% 이상 (주간 이동평균)**
- **경보 기준**: **25% 미만 1주 지속 시 원인 분석 태스크 생성**

### KPI-3. 크래시율
- **정의**: `크래시 세션 수 / 전체 앱 세션 수 * 100`
- **목표**: **1.0% 미만 (일 단위)**
- **경보 기준**: **2.0% 이상 1시간 지속 시 Sev-2 이상으로 승격 검토**

## 4) 운영 채널 정책 명시
- **정책 문구(고정)**: **“신규 기능은 앱만 지원”**
- 적용 위치:
  - 운영자 내부 위키(본 문서)
  - 고객 공지 FAQ
  - 웹 랜딩 페이지 하단 고지
- 운영 원칙:
  - 버그 수정은 앱 우선
  - 웹은 다운로드 유도 및 공지 전달 목적만 유지
  - 기능 요청 접수 시 앱 반영 우선순위로 분류

## 5) 롤백 기준 및 임시 웹 복구
- **기본 원칙**: App-only 정책은 유지하되, **심각 장애(Sev-1)** 시에 한해 임시 웹 복구를 허용
- **임시 웹 복구 발동 조건 (둘 중 하나 이상 충족)**:
  1. 로그인 성공률 **95% 미만**이 **30분 이상** 지속
  2. 크래시율 **5% 이상**이 **30분 이상** 지속
- **복구 범위**:
  - 전체 웹 서비스 복구가 아닌, **핵심 게임 진입/플레이 기능만 임시 복구**
  - 복구 유효시간: **최대 24시간**, 연장 시 Incident Commander 승인 필수
- **종료 조건**:
  - 앱 지표가 경보 기준 이내로 **연속 60분 안정화**
  - 장애 원인 및 재발 방지 항목이 이슈 트래커에 등록 완료
